# ADR-0004-kill-microservices

Date: 2025-09-28

## Status
KILLED

## Decision
Сложная система микросервисов исключена из начальной архитектуры Scarlet Sails.

## Context
В проекте KIT_RnD_Layer4 была попытка создания распределенной архитектуры с множественными микросервисами для различных компонентов системы. Планировались отдельные сервисы для данных, стратегий, исполнения, мониторинга и аналитики.

## Problem
Преждевременная оптимизация привела к избыточной сложности без соответствующих преимуществ. Микросервисы требуют зрелых процессов разработки, мониторинга и деплоя, которых не было в команде. Сетевые взаимодействия между сервисами создавали дополнительные точки отказа.

## Consequences
Scarlet Sails начинается как монолитное приложение с четкой модульной структурой. Переход к микросервисам возможен после достижения 100k строк кода и доказательства необходимости масштабирования. Архитектура проектируется с учетом будущего разделения на сервисы.

## Alternatives Considered
Рассматривался модульный монолит как компромисс, что и было принято в качестве решения. Полные микросервисы отложены до момента реальной необходимости.

## Lessons Learned
Монолит сначала, микросервисы при необходимости. Распределенная архитектура должна решать реальные проблемы масштабирования, а не создаваться "на будущее". Простота архитектуры важнее модности технологий.
